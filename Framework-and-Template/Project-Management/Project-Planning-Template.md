| Project Lead       |               | Oversee project, coordinate team              |
| Developer          |               | Implement features, maintain code             |

## 5. Resources & Tools

* **Code Repository:** GitHub / GitLab link
* **Collaboration Tools:** Slack, Discord, or Teams
* **Task Management:** Jira, Trello, or GitHub Projects
* **Documentation Tools:** MkDocs, Sphinx, Notion
* **Testing & CI/CD:** GitHub Actions, Jenkins, Travis CI

---

## 6. Communication & Collaboration

* **Meeting Schedule:** Weekly / Bi-weekly syncs
* **Channels:** Email, Chat, Discussion Boards
* **Feedback & Review:** Code reviews, pull requests, and peer feedback loops

---

## 7. Quality & Knowledge Management

* **Code Standards:** PEP8, documentation conventions
* **Testing:** Unit tests, integration tests, automated CI checks
* **Knowledge Transfer:** Document processes, tutorials, and lessons learned for future contributors

---

## 8. Sustainability & Well-Being

* **Workload Management:** Assign tasks according to capacity
* **Flexible Participation:** Encourage asynchronous contributions
* **Recognition:** Acknowledge contributions publicly to maintain morale and engagement

---

By using this template, contributors can ensure that open-source projects are **well-planned, collaborative, and sustainable**, while supporting learning, knowledge sharing, and community engagement.
